#### **C. Validation Rules**
```
Data Validation Checks:
1. Format Validation:
   - Dates in correct format
   - Phone numbers valid for country
   - Email format if provided
   - Gender codes (M/F/O only)

2. Business Logic:
   - Age appropriate for class
   - No future birth dates
   - Enrollment date not before birth
   - Class exists in school system
   - Stream exists for class (if provided)

3. Uniqueness Checks:
   - Admission number unique
   - Phone numbers not duplicate (within reason)
   - Email unique (if provided)

4. Completeness:
   - Required fields not empty
   - Parent contact information complete
   - At least one parent phone

Error Reporting:
- Line number of error
- Column name with issue
- Error description
- Suggested fix
- Export errors to CSV for correction
```

#### **D. Processing Engine**
```
Upload Processing Steps:
1. File Parsing:
   - Detect file type
   - Parse with appropriate library
   - Handle encoding issues
   - Convert dates to standard format

2. Validation Phase:
   - Row-by-row validation
   - Collect all errors
   - Group by error type
   - Generate validation report

3. Data Transformation:
   - Map CSV columns to model fields
   - Convert string dates to date objects
   - Format phone numbers
   - Set default values

4. Database Operations:
   - Transaction-based inserts
   - Batch processing for performance
   - Handle duplicates per option
   - Create parent accounts

5. Post-processing:
   - Generate student ID numbers
   - Create parent portal accounts
   - Send welcome messages
   - Update class statistics

Performance Targets:
- 1,000 records in under 30 seconds
- Progress updates every 100 records
- Background processing for large files
- Email notification when complete
```
